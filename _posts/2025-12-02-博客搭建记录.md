## 简介

本博客基于 [Jekyll](https://jekyllrb.com/)，通过 GitHub Pages 托管，实现了静态网站的自动构建与发布。主题采用 Jekyll 官方 Minima，并针对导航、标签、分类等功能做了适配和自定义，采用 Markdown 编写内容，适合长期的技术沉淀与积累。

### 核心步骤

1. **创建 GitHub 仓库并启用 Pages 服务**  
   在 GitHub 上新建仓库，开启 Pages 功能（可自定义域名）。
2. **本地搭建运行环境**  
   安装 Ruby、Bundler 与 Jekyll：
   ```shell
   gem install bundler jekyll
   ```
   初始化站点：
   ```shell
   jekyll new my-blog
   cd my-blog
   ```
3. **本地开发和预览**  
   启动本地服务预览效果：
   ```shell
   bundle exec jekyll serve
   ```
   访问 [http://localhost:4000](http://localhost:4000) 查看页面。
4. **配置站点与主题**  
   编辑 `_config.yml`，定制站点信息，配置插件（如分页、标签等）和主题样式。
5. **内容创作与自定义页面**  
   使用 Markdown 编写文章，按需创建 About、标签页、分类页等自定义页面。
6. **推送至 GitHub 自动部署**  
   将代码推送至仓库，GitHub Pages 会自动完成编译和部署，无需手动操作。

### 参考

> 总结：Jekyll + GitHub Pages 是个人博客极为高效的静态站解决方案，搭建与维护门槛低，内容完全可控，非常适合记录与分享。
